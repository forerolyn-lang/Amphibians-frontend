🧠 Introducción 01

PROBLEMA

Clasificación de casos de vigilancia epidemiológica COVID-19 usando datos categóricos de síntomas (+/-). Se entrenaron dos modelos con el dataset Surveillance.csv del UCI Repository.

La API Flask desplegada en Render.com sirve predicciones en tiempo real. El frontend (HTML) puede estar en GitHub Pages.

  • DatasetSurveillance.csv (UCI)
  • Modelo AMLPClassifier (50,30,10)
  • Modelo BLogisticRegression C=500
  • BackendFlask + Gunicorn
  • Serializaciónpickle (.pkl)
  • Deploy APIRender.com (free)
  • Deploy WebGitHub Pages

🚀 Guía de Deploy en Render 07
1
Entrenar y guardar los modelos localmente
Coloca Surveillance.csv junto a train_and_save.py y ejecuta: python train_and_save.py. Generará models/mlp_model.pkl, mlp_scaler.pkl, lr_model.pkl, lr_scaler.pkl.
2
Subir el proyecto a GitHub
Crea un repositorio en GitHub y sube la carpeta covid-api/ con: app.py, requirements.txt, render.yaml y la carpeta models/ con los 4 archivos .pkl.
3
Crear el Web Service en Render
Ve a render.com → New → Web Service → conecta tu repositorio. Build Command: pip install -r requirements.txt. Start Command: gunicorn app:app --bind 0.0.0.0:$PORT. Plan: Free.
4
Obtener la URL de Render y configurar el frontend
Render te dará una URL como https://covid-ml-api.onrender.com. Pégala en el campo de la parte superior de esta página y presiona Conectar.
5
Subir el frontend a GitHub Pages
Coloca este index.html en tu repositorio (rama gh-pages o carpeta docs/). GitHub Pages lo servirá gratis. La URL de la API ya estará guardada en el código.

Endpoints de la API

  • GET /Health check — verifica que la API esté activa
  • POST /predict/nnPredicción individual — Red Neuronal
  • POST /predict/lrPredicción individual — Regresión Logística
  • POST /predict/batchPredicción por lotes — con distribución y matriz de confusión
